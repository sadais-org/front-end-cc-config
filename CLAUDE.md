- 使用中文回复
- 无论任务复杂度如何、工具调用链多长，所有面向用户的输出（包括分析、说明、标题）必须全程使用中文，不得因内容为技术性质而切换为英文
- 始终使用 Context7 MCP 获取最新的文档
- 对于 GitHub 相关的操作，应该使用 gh 命令行工具而不是 WebFetch

## 工作规范
- 在编写任何代码之前，请先描述你的方案并等待批准，如果需求不明确,务必先提出澄清问题，使用交互式访问向用户确认
- 用户未明确回复"确认"、"可以"、"ok"等同意信号时，禁止执行任何文件修改操作；工具加载消息（如"Tool loaded."）不属于确认信号
- 如果一项任务需要修改超过 3 个文件，请先停下来，将其分解成更小的任务
- 分析/修复问题时，先梳理文件依赖树，通读全部相关文件后再处理。
- **必须使用 AskUserQuestion 工具进行交互式确认，禁止用纯文字提问代替。** 当流程中需要用户做出选择或确认时（如是否执行某操作、确认提交计划、选择处理方式等），必须调用 `AskUserQuestion` 工具提供选项让用户点选，不得仅在输出中文字提问后直接继续执行。

## 用户决策遵循规范
- 用户通过 AskUserQuestion 做出的选择（点选选项或自定义输入）具有**最高优先级**，等同于直接指令，必须 100% 严格遵循
- **禁止**在收到用户选择后，根据上下文或其他规则自行"优化"、"调整"或"覆盖"用户的选择
- 当用户选择与其他规则冲突时，**用户选择优先**，但必须在执行前用 AskUserQuestion 明确告知用户存在冲突并确认
- 执行用户选择时，必须逐字理解用户输入的自定义内容，不得擅自替换、扩展或缩写用户明确指定的内容
- 如果对用户选择的理解存在歧义，必须用 AskUserQuestion 再次确认，禁止自行推断

## 脚本执行规范
- 执行脚本命令时时刻谨记当前所处上下文目录，禁止在错误的上下文目录执行命令

## 清理规范
- 任务完成后，必须删除执行过程中创建的所有临时文件
- 临时脚本执行完毕后立即删除
- 创建临时文件时需告知用户文件路径，便于追踪和清理
