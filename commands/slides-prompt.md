# 幻灯片提示词生成器

根据用户提供的文档，自动分割成多页幻灯片提示词，用于 NotebookLM 或其他 AI 工具生成演示文稿。

## 工作流程

### 步骤 1：确定文档

如果用户提供了文档路径 `$ARGUMENTS`，读取该文档。
如果没有提供，询问用户要处理哪个文档。

### 步骤 2：选择主题

使用 AskUserQuestion 工具让用户选择主题：

```
请选择幻灯片主题风格：
1. Ocean Depths - 深海蓝调，专业沉稳（推荐）
2. Arctic Frost - 冰川白蓝，清新简洁
3. Botanical Garden - 植物绿意，自然清新
4. Modern Minimalist - 现代极简，干净利落
```

### 步骤 2.5：选择每组页数

使用 AskUserQuestion 工具让用户选择每组生成的页数：

```
请选择每组提示词生成的页数：
1. 10 页（推荐，适合大多数文档）
2. 5 页（较短内容）
3. 8 页（中等内容）
4. 15 页（超长章节）
```

将用户选择的页数记为 `PAGE_PER_GROUP`，后续步骤中使用此变量。

### 步骤 3：分析文档结构并计算页数

读取文档后，**根据内容量动态计算总页数**：

1. 识别所有章节标题（# ## ### 等）
2. 分析每个章节的内容复杂度：
   - 代码块数量
   - 步骤/列表项数量
   - 子章节数量
   - 配置/命令数量

3. **动态分配页数规则**：
   | 章节类型 | 页数 |
   |----------|------|
   | 简单概念介绍（无代码） | 1 页 |
   | 包含 1-2 个代码块/命令 | 1-2 页 |
   | 包含 3-5 个步骤或代码块 | 2-3 页 |
   | 复杂章节（多个子章节、大量代码） | 3-5 页 |
   | 封面页 | 1 页 |
   | 目录页 | 1 页 |
   | 总结/资源页 | 1-2 页 |

4. **超长章节自动分割**：
   - 如果某章节计算出的页数 > `PAGE_PER_GROUP`，自动分割成多个部分
   - 分割命名规则：「章节名（一）」「章节名（二）」「章节名（三）」...
   - 每个分割部分的页数 ≤ `PAGE_PER_GROUP`
   - 按子章节或逻辑单元进行分割，保持内容完整性
   - 示例：「自定义命令与技能」20 页 → 分割为「自定义命令与技能（一）」10 页 + 「自定义命令与技能（二）」10 页

5. 计算总页数 = 封面 + 目录 + 各章节页数之和 + 总结
6. 根据 `PAGE_PER_GROUP` 计算需要多少组提示词
7. 规划每组提示词覆盖的章节（分割后的部分各占一组）

### 步骤 4：确认分割方案

使用 AskUserQuestion 展示**动态计算的分割方案**，让用户确认或调整：

```
文档分析结果：
- 总章节数：X 个
- 动态计算总页数：Y 页
  - 封面 + 目录：2 页
  - 章节 1「XXX」：N 页（含 M 个代码块）
  - 章节 2「XXX」：N 页（含 M 个步骤）
  - 章节 3「XXX」：超长，已分割为：
    - 「XXX（一）」：10 页
    - 「XXX（二）」：10 页
  - ...
  - 总结 + 资源：2 页
- 需要 Z 组提示词（每组 PAGE_PER_GROUP 页）

分割方案：
- 第 1 组（第 1-10 页）：封面 + 目录 + 章节 1-2
- 第 2 组（第 11-20 页）：章节 3「XXX（一）」
- 第 3 组（第 21-30 页）：章节 3「XXX（二）」
- ...

是否确认此方案？可调整各章节页数。
```

### 步骤 5：读取主题配置

根据用户选择的主题，从 `~/.claude/skills/theme-factory/themes/` 读取对应的主题配置文件，提取：
- 配色方案（背景色、强调色、文字色等）
- 字体配对（标题字体、正文字体）
- 风格描述

### 步骤 6：生成提示词

为每组生成格式化的提示词，**必须为每页指定明确的带序号标题和详细内容要求**：

```markdown
## 提示词 X/Y（第 N-M 页）

📁 **生成文件命名：** `[文档名]-幻灯片-第X组.pptx`

请根据上传的文档生成幻灯片，使用 [主题名] 主题。

⚠️ 重要提示：
1. 请严格使用我指定的标题，不要自行更改！
2. 必须包含文档中的具体代码、命令、配置示例，不要只写概述！
3. 每页内容要详细完整，不要省略关键步骤！

🎨 主题：[主题名]（[背景色] 背景，[强调色] 强调，[文字色] 文字）

生成以下 PAGE_PER_GROUP 页幻灯片：

【第 N 页】标题：[明确的中文标题]
📍 对应文档章节：[章节名称]
📋 必须包含的内容：
- [要点 1]
- [要点 2]
🔑 关键细节（必须展示）：
- 代码/命令：`[具体代码或命令]`
- 配置示例：[如有]

【第 N+1 页】标题：[明确的中文标题]
📍 对应文档章节：[章节名称]
📋 必须包含的内容：
- [要点 1]
- [要点 2]
🔑 关键细节（必须展示）：
- 代码/命令：`[具体代码或命令]`

...
```

**提示词格式要求**：
- 使用【第 X 页】标题：格式，强制 NotebookLM 使用指定标题
- 📍 标注对应的文档章节，便于 AI 定位内容
- 📋 列出必须包含的要点（3-5 个）
- 🔑 明确列出关键细节：代码、命令、配置、步骤等
- 添加 ⚠️ 警告提示，强调不要省略细节

### 步骤 7：输出结果

1. **终端输出**：显示所有生成的提示词
2. **保存文件**：将提示词保存到 `[文档名]-slides-prompts.md`

## 输出文件格式

```markdown
# [文档名] 幻灯片提示词

## 主题：[主题名]

### 主题配置
| 元素 | 颜色/字体 | 用途 |
|------|-----------|------|
| ... | ... | ... |

---

## 提示词 1/N（第 1-PAGE_PER_GROUP 页）

📁 **生成文件命名：** `[文档名]-幻灯片-第1组.pptx`

...

## 提示词 2/N（第 PAGE_PER_GROUP+1 至 2×PAGE_PER_GROUP 页）

📁 **生成文件命名：** `[文档名]-幻灯片-第2组.pptx`

...
```

## 注意事项

1. 每组提示词页数由用户选择（PAGE_PER_GROUP），默认 10 页
2. **必须使用【第 X 页】标题：格式指定每页标题**
3. **必须添加文件命名标识**：`📁 **生成文件命名：** [文档名]-幻灯片-第X组.pptx`
4. 添加 ⚠️ 警告防止 AI 自行更改标题或省略细节
5. **每页必须标注对应的文档章节**，便于 AI 定位内容
6. **必须列出关键细节**：代码、命令、配置、步骤等，防止内容遗漏
7. 第一组始终包含封面页和目录页
8. 最后一组包含总结页和资源页
9. 标题使用简洁中文，便于整合时识别顺序
